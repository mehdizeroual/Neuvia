Tu es un développeur senior spécialisé en front-end moderne, UX institutionnelle et design futuriste.

Objectif :
Créer un site web de présentation + démonstration pour "Neuvia / Lirra", une plateforme IA de construction de mondes vidéo éducatifs, co-construite avec l’Éducation nationale.

Le site n’est PAS un produit final mais une démonstration fonctionnelle crédible destinée à des acteurs institutionnels, éducatifs et investisseurs.

STACK & TECH :
- Stack libre mais moderne (React / Next.js recommandé)
- Pas de backend réel
- Données mockées uniquement
- Navigation fluide, animations légères et modernes

DIRECTION ARTISTIQUE / DESIGN :
- Design futuriste, épuré, institutionnel
- Le site doit évoquer clairement “le futur de l’éducation”
- Palette de couleurs :
  - Bleu turquoise / bleu clair (couleur dominante)
  - Violet (accent secondaire, innovation / IA)
  - Blanc et noir pour la lisibilité des textes
- Typographie moderne, lisible, légèrement technologique
- Utiliser des effets subtils :
  - Dégradés bleu → violet
  - Légères animations au hover
  - Cartes avec ombres douces ou effet glassmorphism léger
- L’interface doit rester sérieuse et crédible (pas gaming)

AUTHENTIFICATION :
- Page /login
- Champs email + mot de passe
- Aucune vérification réelle
- Tout couple email/mot de passe permet l’accès
- Objectif : simuler un environnement sécurisé

STRUCTURE APRÈS LOGIN :
Page principale type dashboard appelée "Library"

ONGLET 1 : "Expériences vérifiées"
- Expériences éducatives co-construites avec l’Éducation nationale
- Filtres :
  - Matière : Histoire, Sciences, Mathématiques
  - Niveau : 3ème, Seconde, Première, Terminale
- Cartes contenant :
  - Titre
  - Description courte
  - Matière
  - Niveau
  - Badge visible "Validée Éducation nationale"
  - Bouton "Découvrir" ou "Lancer l’expérience" (mock)

ONGLET 2 : "Expériences de la communauté"
- Expériences créées par les utilisateurs
- Même logique de filtres
- Cartes avec :
  - Nom de l’expérience
  - Auteur fictif
  - Tags pédagogiques
  - Statistiques d’utilisation fictives
  - Badge "Communauté" ou "En attente de validation"

PAGES STATIQUES OBLIGATOIRES :
- Mentions légales
- Politique de confidentialité
- Conditions générales d’utilisation
- Politique cookies
- À propos
- Contact

FOOTER :
- Présent sur toutes les pages
- Liens vers toutes les pages légales
- Design discret, institutionnel

BONUS UX / CRÉDIBILITÉ :
- Page "Comment ça marche ?"
- Page "Vision pédagogique"
- Sections expliquant :
  - Le rôle de l’IA dans l’apprentissage
  - La complémentarité avec les enseignants
  - L’alignement avec les programmes scolaires

Tout doit être fonctionnel visuellement, fluide, cohérent et crédible,
même si toutes les données et actions sont simulées.
